Overview of the World Wide Web

The World Wide Web(WWW), commonly known as the web, is a vast and interconnected system of information and resources that can be accessed through the internet.

Concept Description
Concept of the Web The World Wide Web and the Internet are not the same thing. The internet is the global network infrastructure that enables communication between computers, while the web is an application that uses the internet to provide a user-friendly way to access and share information.
Web Pages and Websites The fundamental unit of the web is a web page, which is a document containing text, images, multimedia, and hyperlinks. Web pages are hosted on web servers and can be accessed through web browsers. A collection of related web pages forms a website.
URL (Uniform Resource Locator) URLs are addresses that uniquely identify resources on the web. They consist of a protocol identifier (e.g., HTTP or HTTPS), a domain name (e.g., www.youtube.com), and a path to the specific resource.
HTTP (Hypertext Transfer Protocol) HTTP is the protocol used for transferring web pages and other resources over the internet. HTTPS (Hypertext Transfer Protocol Secure) is a more secure version that encrypts data between the user’s browser and the Web Servers, ensuring privacy and security.

History of the Internet

1950s–1960s: The Foundation

1957: The Soviet Union launches Sputnik, prompting the U.S. to establish the Advanced Research Projects Agency (ARPA) to advance scientific and technological research.

The late 1950s to early 1960s: ARPA funds research into computer networking, leading to the development of packet-switching techniques.

1969: ARPANET, the precursor to the internet, is established by connecting four universities using packet-switching technology. This event is considered the birth of the internet.

The 1970s: Early Networking and Protocols

1971: The first email program is developed by Ray Tomlinson, allowing messages to be sent between users on ARPANET.

1973: The first international connections to ARPANET are established, extending its reach beyond the United States.

1974: The Transmission Control Protocol (TCP) is proposed by Vinton Cerf and Bob Kahn, providing a foundation for reliable data transmission.

The 1980s: TCP/IP and Domain Names

1983: ARPANET adopts the TCP/IP protocol suite, unifying various networks into a single network of networks, laying the foundation for the modern internet.

1983: The domain name system (DNS) is introduced, enabling the use of human-readable addresses (e.g., www.example.com) instead of numerical IP addresses.

1989: Tim Berners-Lee proposes the concept of the World Wide Web, envisioning a system of interconnected hypertext documents accessible through the internet.

The 1990s: The Rise of the World Wide Web

1991: The first web page is created by Tim Berners-Lee, and the first web browser, Worldwide Web (later renamed Nexus), is developed.

1993: The graphical web browser Mosaic is released, popularizing web browsing and making the web more accessible to the public.

1994: Commercial interests begin to invest in the internet, leading to the creation of online shopping and e-commerce.

1995: The NSF lifts restrictions on commercial use of the internet, further accelerating its growth and commercialization.

1998: Google is founded by Larry Page and Sergey Brin, revolutionizing online search with its advanced search algorithms.

The 2000s: Internet Expansion and Social Media

Early 2000s: The dot-com bubble bursts, leading to the reevaluation of internet business models.

2004: Facebook is launched, marking the beginning of the social media era, and changing how people interact online.

2007: Apple introduces the iPhone, sparking the smartphone revolution and increasing internet access through mobile devices.

2009: Bitcoin, the first decentralized cryptocurrency, is introduced, paving the way for blockchain technology.

The 2010s: Mobile Internet and Beyond

The 2010s: Mobile internet usage surges with the proliferation of smartphones and tablets.

2013: Edward Snowden's revelations about mass surveillance highlight concerns about online privacy and data security.

2016: The Internet of Things (IoT) gains traction, connecting everyday objects and devices to the internet.

2019: The rollout of 5G technology begins, promising faster and more reliable internet connectivity.

2020s and Beyond: Ongoing Evolution

The internet continues to evolve with advancements in artificial intelligence, augmented reality, virtual reality, and more.

WWW vs the Internet

The Internet:

The Internet is a vast global network of interconnected computer networks. It's a physical infrastructure that allows computers and devices to communicate with each other using a standardized set of protocols. These protocols govern how data packets are transmitted, routed, and received across the network. The Internet provides the underlying framework for various services, applications, and technologies, including email, file sharing, online gaming, video conferencing, and much more.

Key points about the Internet:

  • It's a network of networks, connecting millions of computers, servers, and devices worldwide.
  • It functions as a backbone for various online services and applications.
  • The Internet enables data exchange and communication across different devices and platforms.

World Wide Web (WWW or Web):

The World Wide Web is a specific application or service that runs on top of the Internet. It's a collection of interconnected hypertext documents (web pages) that are accessible via the Internet. The Web enables users to access and navigate these documents using web browsers, and it's characterized using hyperlinks that connect web pages to each other. The Web allows users to view text, images, videos, and other multimedia content hosted on web servers.

Key points about the World Wide Web:

  • It's a subset of the broader Internet, providing a way to access and view content.
  • Web pages are created using languages like HTML (Hypertext Markup Language).
  • Web browsers (e.g., Chrome, Firefox, Safari) are used to access and interact with web pages.
  • Hyperlinks allow users to navigate between different web pages and websites.

Evolution of the Internet

1960s: Birth of the ARPANET

  1. The Advanced Research Projects Agency Network (ARPANET) is established by the U.S. Department of Defense's Advanced Research Projects Agency (ARPA).
  2. ARPANET links four major research universities and uses packet-switching technology to exchange data.

1970s: Emergence of TCP/IP and Email

  1. The Transmission Control Protocol (TCP) and the Internet Protocol (IP) are developed, forming the foundation of the modern Internet.
  2. Ray Tomlinson invents email, allowing users to send messages between different computers.

1980s: TCP/IP Adoption and Domain Names

  1. TCP/IP becomes the standard networking protocol, leading to the development of a unified global network.
  2. The domain name system (DNS) is introduced, simplifying the process of accessing websites by using human-readable addresses.

1990s: Rise of the World Wide Web and Commercialization

  1. Tim Berners-Lee invents the World Wide Web, introducing hypertext documents accessible through web browsers.
  2. Web browsers like Mosaic and Netscape Navigator make the Web user-friendly and accessible to the public.
  3. E-commerce, online shopping, and the dot-com boom drive commercialization of the Internet.
  4. Search engines like Yahoo, AltaVista, and Google emerge, making it easier to find information online.

2000s: Web 2.0 and social media

  1. The concept of Web 2.0 gains prominence, emphasizing user-generated content, social networking, and interactivity.
  2. Blogs, social media platforms (e.g., Facebook, Twitter), and video-sharing sites (e.g., YouTube) reshape online communication.
  3. Broadband Internet becomes more widespread, enabling faster and richer online experiences.

2010s: Mobile Internet and Cloud Computing

  1. Smartphones and mobile devices lead to the rapid growth of mobile Internet usage and app-based services.
  2. Cloud computing becomes mainstream, allowing users to store data and access applications remotely.
  3. Internet of Things (IoT) devices, like connected home devices and wearables, contribute to the expansion of Internet-connected devices.

2020s and Beyond: Advanced Technologies and Connectivity

  1. The rollout of 5G technology promises even faster and more reliable wireless connectivity.
  2. Artificial intelligence (AI), virtual reality (VR), augmented reality (AR), and blockchain technology shape new online experiences.
  3. Privacy and cybersecurity concerns prompt discussions about data protection and digital rights.

Internet Protocols

Protocol Description
Transmission Control Protocol (TCP) TCP is a connection-oriented protocol responsible for ensuring reliable and ordered data transmission between devices over a network. It breaks data into packets, sends them, and ensures they arrive at the destination in the correct order. If a packet is lost, TCP requests a retransmission. TCP guarantees data integrity and is commonly used for applications that require accurate data delivery, such as web browsing and email.
Internet Protocol (IP) IP is the principal protocol responsible for routing data packets between devices across different networks. It assigns unique IP addresses to devices and handles the addressing and routing of packets so that they reach their intended destinations. There are two main versions of IP: IPv4 (with 32-bit addresses) and IPv6 (with 128-bit addresses) to address the growing demand for unique addresses due to the increasing number of connected devices.
File Transfer Protocol (FTP) FTP is used for transferring files between computers on a network. It provides a way to upload and download files from remote servers and is often used for website maintenance and data exchange.
Simple Mail Transfer Protocol (SMTP) SMTP is used for sending emails, while POP3 and IMAP are used for receiving emails from a mail server. SMTP transfers outgoing mail, while POP3 and IMAP allow users to retrieve and manage their emails from the server.

Web Application Architecture

Web application architecture refers to the structural design of a web application, outlining how its components interact and work together to deliver functionality to users. There are two primary components in web application architecture: the client-side and the server-side.


Description
Client-Side The client-side of a web application is responsible for the presentation and user interface. It includes the web browser on the user's device (desktop, mobile, tablet), which interacts with the server-side to request and display content. Client-side technologies include HTML (for structuring content), CSS (for styling), and JavaScript (for adding interactivity and dynamic behavior to web pages). Modern client-side development often involves frameworks like React, Angular, or Vue.js, which provide tools for building sophisticated user interfaces.
Server-Side The server-side of a web application handles the business logic, data processing, and communication with databases. It includes the web server that hosts the application, processes requests from clients, and delivers responses. Server-side technologies include programming languages like Python, Ruby, PHP, Java, and frameworks like Ruby on Rails, Django, Laravel, Spring Boot, etc. The server-side interacts with databases, external APIs, and other services to process data and generate content dynamically.

Client-side and server-side engineering


Client-Side Engineering:

Client-side engineering involves developing the front-end components of a web application that users interact with directly. Here are key aspects of client-side engineering:


  1. User Interface Development:
    • Creating responsive and visually appealing user interfaces using HTML, CSS, and JavaScript.
    • Ensuring that web pages are compatible with various devices and screen sizes.
  2. Interactivity and User Experience:
    • Enhancing user experience through dynamic features, animations, and real-time updates using JavaScript and front-end libraries/frameworks.
    • Implementing user interactions like form submissions, clicks, and navigation.
  3. Web Browser Compatibility:
    • Ensuring that web applications function correctly across different web browsers and versions.
    • Addressing browser-specific issues and ensuring consistent behavior.

Server-Side Engineering:

Server-side engineering involves building the back-end components of a web application that handle data processing, business logic, and interactions with databases and external services. Here are key aspects of server-side engineering:


  1. API Design and Development:
    • Creating APIs (Application Programming Interfaces) that allow client-side and other applications to communicate with the server.
    • Designing endpoints for retrieving, creating, updating, and deleting data.
  2. Database Interaction:
    • Designing database schemas and models to store and organize data.
    • Writing queries and managing data interactions using database management systems like MySQL, PostgreSQL, MongoDB, etc.
  3. Security and Authentication:
    • Implementing security measures to protect sensitive data and prevent unauthorized access.
    • Managing user authentication and authorization using techniques like OAuth, JWT, and encryption.
  4. Performance and Scalability:
    • Optimizing server-side code for efficient data processing and response times.
    • Designing for scalability to accommodate increasing user loads and traffic.
